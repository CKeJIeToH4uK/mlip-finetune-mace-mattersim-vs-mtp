## Learning curve finding

Robust MoNbTaVW/alloy learning traces were found without using raw datasets/checkpoints/model weights:

- MatterSim: `repo/coursework/new_datasets/alloy/mattersim/train.log` contains per-epoch train/validation loss, MAE(e), and MAE(f) for epochs 0-199.
- MACE-EW50: `repo/coursework/new_datasets/alloy/mace_ablation_normal_ew10_ew50_20260509/full_ew50_normal/results/alloy_mace_full_ew50_normal_run-42_train.txt` contains JSON-lines validation metrics at evaluated epochs.

The generated learning figures plot training-log validation Energy MAE and Forces MAE traces for optimization sanity checks only. They are not final-evaluation ranking plots: the MatterSim training-log force-MAE trace is not numerically aligned with the final canonical MatterSim validation metric used in the main comparison. MTP stdout traces were not plotted as learning curves because the available MLIP iteration traces do not provide a clean validation metric series with slide-ready semantics.

## Caveats

- Alloy means MoNbTaVW in these outputs.
- MACE-EW50 is not default MACE; it is an energy-weighted fine-tuning protocol.
- MACE default uses the fair lr=0.001 fair200 default-loss baseline in the canonical CSV. The old lr=0.01 default baseline exists with valid Energy MAE 34.744, Energy RMSE 64.448, Forces MAE 70.603, Forces RMSE 123.247, but is not used as the preferred default row.
- Do not aggregate ablations as an ensemble; each row is a separate completed protocol.
- Do not show mean +/- std; these are single completed runs, not confirmed repeated runs.
- MTP-16 audit note: `metrics.json`, model basis length, and parameter count support actual level 16; the conflicting log line remains report-only.
- MACE-EW50 outperforms MTP-20 on all four listed validation metrics, while MatterSim remains best on Energy MAE/RMSE in the verified table.
