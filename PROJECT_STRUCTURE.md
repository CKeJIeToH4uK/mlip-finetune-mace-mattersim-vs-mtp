# Структура репозитория

`coursework_clean/` сохраняет проверяемую часть исследовательского проекта: результаты, код проверки, протоколы экспериментов, графики, provenance и архивный контекст. Тяжелые данные, веса и сборочные артефакты вынесены в manifest-файлы.

## Корень

- `README.md` — назначение репозитория и быстрый старт.
- `REPRODUCIBILITY.md` — что можно проверить локально и какие активы намеренно исключены.
- `requirements.txt` — окружение для проверки результатов, работы со сводными таблицами и графиками; файл не обещает полного переобучения моделей.

## Основные папки

- `docs/` — постановка задачи, описание датасетов, моделей, метрик, результатов, ограничений и происхождения результатов.
- `results/verified_metrics/` — проверенные CSV-файлы с метриками по отдельным системам и производные аннотированные таблицы.
- `results/appendix_tables/` — дополнительные таблицы для приложения и расширенные протокольные результаты.
- `results/sanity_checks/` — дополнительные проверочные таблицы, включая MTP ensemble для organic 300K.
- `results/figures/` и `report_assets/figures/` — отобранные графики для просмотра и будущего отчета.
- `provenance/handoff_tables/` — вспомогательные handoff-таблицы; они не являются полным источником финальных результатов.
- `provenance/source_mirror/` — историческое зеркало исходных скриптов, сохраненное только для сверки происхождения результатов.
- `provenance/` — сводки, фрагменты логов, slurm scripts, raw eval refs, manifests и checksums.
- `src/mlip_benchmark/` — легкие вспомогательные модули в `analysis/`, `evaluation`, `plotting` и `utils`.
- `scripts/` — проверки clean repo и вспомогательные скрипты.
- `presentation_archive/` — финальная презентация и выбранные preview/backup материалы.
- `archive/` — старый контекст и исключенные ветки как история, не источник финальных чисел.
