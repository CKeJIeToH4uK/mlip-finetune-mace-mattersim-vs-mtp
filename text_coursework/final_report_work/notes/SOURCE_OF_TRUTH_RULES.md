# Правила источника истины

## Финальные числа

- Финальные численные значения брать только из `../text_coursework/RESULTS_SOURCE_OF_TRUTH.md` относительно корня clean repo и verified CSV.
- Внутри clean repo verified CSV находятся в `results/verified_metrics/`.
- Перед переносом чисел в текст, таблицы или подписи графиков проверять систему, модель, разбиение, метрику и единицы.

## Запрещенные подмены источников

- Не использовать старые handoff-таблицы как главный источник чисел.
- `provenance/handoff_tables/` можно использовать только для сверки происхождения результатов.
- Графики и markdown-превью не заменяют verified CSV и `RESULTS_SOURCE_OF_TRUTH.md`.

## Термины и модели

- Не путать `MACE default` и `MACE-EW50`; это разные строки результатов.
- Для `H2O` и `MoNbTaVW` писать `valid` или `validation`, не `test`.
- Не утверждать, что универсальные модели всегда лучше.
- Формулировать выводы по конкретной системе, разбиению и метрике.

## Границы выводов

- Не делать выводы о runtime/cost-efficiency.
- Не добавлять утверждения о долгой стабильности MD-траекторий без отдельного подтвержденного источника.
- Не переносить в финальный текст числа, которые не проходят проверку по источнику истины.
